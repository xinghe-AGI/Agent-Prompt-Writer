# Prompt Method

本文件是 AI Prompt Writer 的核心方法。不要再按很多框架拆文件；先判断 prompt 的用途，再选最小可用结构。

## 1. 先判断用途

| 用途 | 写法 |
|---|---|
| 一次性任务 | 短 prompt，保留角色、任务、约束、输出格式 |
| 结构化输出 | 使用完整骨架，重点写字段契约 |
| JSON 抽取 | 强制唯一输出形态、字段类型、缺失值、枚举和来源约束 |
| Agent 执行 | 写清目标、范围、允许动作、禁止动作、检查点、停止条件 |
| 自动化工作流 | 写清触发、步骤、字段映射、失败处理、最终产物 |
| 长期角色 | 加入简介、版本、初始化、交互规则和维护纪律 |

如果用途不清楚，最多问 3 个问题。优先问会影响输出契约的问题，例如目标工具、输出形态、字段、异常输入。

## 2. 统一核心骨架

生产级 prompt 使用这个骨架：

```text
# 角色 : <助手名称>

## 背景 :
说明输入来源、使用场景、模型易错点。

## 简介 :
- author:
- version:
- language:
- description: <输入 + 输出 + 本版关键变化>

## 任务 :
分点说明模型要完成的动作。

## 约束条件（重要）:
- 输出形态
- 缺失值策略
- 字段来源约束
- 枚举和类型约束
- 异常输入处理

## 技能 :
说明模型需要执行的识别、判断、归一化、分类能力。

## 工作流 :
1. ...
2. ...
3. ...

## 输出格式（必遵）:
定义字段、类型、枚举、顺序。

## 输入模板 :
告诉用户如何提供输入。

## 输出样例 :
至少覆盖成功样例和异常样例。

## 初始化 :
简短说明如何开始。
```

简单任务不要强行套完整骨架。压缩为：

```text
角色：
任务：
输入：
约束：
输出：
完成标准：
```

## 3. 9 条硬标准

1. **简介必须可追踪版本变化**：`description` 写清输入、输出、关键字段和本版变化。
2. **输出形态只能有一个**：JSON、Markdown、自然语言、表格、代码补丁只能选一种主形态。
3. **字段契约要全量闭环**：简介、约束、输出格式、样例、初始化中的字段口径要一致。
4. **缺失值策略要统一且可执行**：无法识别时优先返回 `null`，不要编造默认值。
5. **枚举值要可列举、可校验**：分类字段必须给允许值集合。
6. **字段来源约束是防幻觉关键**：写清哪些字段必须来自输入，哪些字段允许模型判断。
7. **规则写法用“条件 + 结果”**：少写“请准确”，多写“若 X，则输出 Y”。
8. **样例要覆盖主流程和异常流程**：至少有成功样例；生产级任务补部分缺失和非目标输入样例。
9. **初始化要短，但必须能直接开工**：重申身份、输出结构和用户输入方式即可。

## 4. 辅助原则

论文 26 条原则只作为轻量检查，不机械套用。默认采用这些稳定原则：

- 明确受众。
- 使用正向指令。
- 用标题、分隔符或代码块分开指令、输入、示例和输出。
- 示例比抽象规则更能锁定格式。
- 复杂任务拆步骤。
- 需要澄清时先提问。
- 输出格式必须具体。
- 不必要的客套、激励和惩罚语句默认不用。

## 5. Agent Prompt 最小模板

```text
目标：

当前状态：

允许范围：

禁止动作：

执行步骤：

检查点：

停止并询问：

完成标准：
```

Agent prompt 必须写清边界。凡是删除文件、改依赖、改数据库 schema、发送消息、提交表单、购买、外部发布，都要加入人工确认。

## 6. JSON 抽取 Prompt 最小模板

```text
你是一个信息抽取助手。你的任务是只基于输入内容抽取结构化 JSON。

约束：
- 只输出一个合法 JSON 对象。
- JSON 外不要输出任何解释、Markdown 或代码块。
- 所有字段必须来自输入，除非字段说明允许判断。
- 无法识别的字段返回 null。
- 枚举字段只能使用给定枚举，无法判断返回 null。

字段：
- field_name: 类型，来源，缺失值策略，枚举值（如有）

输入：
<<<
[用户输入]
>>>

输出格式：
{
  "field_name": null
}
```

## 7. 最终自检

交付前只做这几个检查：

- prompt 是否能直接复制使用。
- 任务是否是具体动作。
- 输入和输出边界是否清楚。
- 输出形态是否唯一。
- 字段、枚举、缺失值、来源约束是否完整。
- Agent 是否有范围、禁止动作、检查点和停止条件。
- 有没有密钥、token、cookie、账号信息。
- 有没有为了显得高级而套了不必要框架。
